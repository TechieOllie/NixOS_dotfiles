# hosts/

**Phase:** 1 (Foundation) — new host directories are added as the project grows.

Each subdirectory is one physical or virtual machine. A host describes **what
machine is being built**, not how any feature works internally — it should
stay small: a couple of data files and a handful of imports.

A host directory holds:

- `default.nix` — wires identity (`variables.nix`) into the system and lists
  which profile(s)/modules it imports.
- `variables.nix` — machine identity (hostname, timezone, keymap, user).
- `features.nix` — this machine's capability overrides (`config.features.*`).
- `disko.nix` — declarative disk layout.
- `hardware-configuration.nix` — generated by NixOS, never hand-edited.
- `secrets.nix` + `secrets/secrets.yaml` — this host's sops-nix wiring and
  encrypted secrets, once it's bootstrapped.

A host only gets a `nixosConfigurations` entry in `flake.nix` once it has
real `hardware-configuration.nix`/`secrets.nix` — see `CLAUDE.md` for which
hosts are bootstrapped right now.

Full rationale: [`NixOS-Configuration-Guide.md`](../NixOS-Configuration-Guide.md).
